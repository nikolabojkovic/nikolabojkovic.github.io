Quality on the budget

First we need the budget, then we can allocate resources, set deadlines and do the planning. Of cource there has to be balance between quality and how fast we are goint to build a software.

Building a software requires a lot of efort and time. Some parts of the system can not be built in parallel and sometimes it is not possible to have more then one person doing one feature at a time. So we have to balance between how fast are we going to build our software and keep the quality of the software at a high level.

Rushing trough the proccess can create a huge problems later when we realize that because of pure quality of the software we have to re-write it from scratch. Is that more expensive then writing high quality software from start?

It is not a rare case that software has to be re-writen because of bad code quality (spaghetty code) and that is the reason why we need to have at least decent quality of the software and decent development speed.
