Monolith to microservices

When software reaches stage where monolith has to be splitted accros multiple domains, first we have to separate domains into different bounded contexts and then farther into microservice. Bounded context is logical and micrroservice phisical separation between domains.

Process of separatioin is time consuming and expensive as well, but benefits are huge.

There are serveral phases during this conversion:

Domain separation

Code refactoring

Testing

Domain isolation

Components/Microservices Integration

Infrastructure refactoring

Optimization

Scaling
