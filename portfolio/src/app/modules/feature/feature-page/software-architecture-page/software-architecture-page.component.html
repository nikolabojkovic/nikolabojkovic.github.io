Software Architecture

It all depends of the complecity of the project. If we know that project is going to be complex and we now a lot about the project in advance then we can choose more complex architecture from start, otherwise we can start samll, start with simle three-tier architecture and then refactor to a more complex architecture later in the process.

Every architecture has pros and cons. Microservices are great for huge systems and complex domains. Clean architecture is great way to decouple the sytem and organize files and folder. Three-tier architecture is great for small apps and apps that has to be developed really fast.

On the other hand microservices are complex architecture and take a lot of time to implement. Clean architecture is overkill for small app and three-tier achitecture is not good enough to suport medium size and big applications.

It could happen that we have to switch to a different achitecture in developement process and that is fine. There is no one universal solution for everyting, it all depends what wee need and what kind of problem we are trying to solve.
