Development Strategy

First we need to know what kind of application we are going to build. Is it going to be a small app, medium size app or huge system used by a lot of uses? Is it going to be a monolith app or we need to split the domain accross microservices. It all depends.

How many team members do wee need to have? What about other resources?

A lot of questions has to be answered before we start development. As soon as we get all the answers we have to analyse everyting and make decisions.

What kind of strategy we are going to use depends on complecity of the project, budget of course and how much time do we have to develop a solution.

Then we have to choose, are we going to develop in interation or we going to have a straight forward plan and do it all in one go.

Usually it is the best idea to start small as a simple app and then grow and develop to a more complicated system if we need to.
